# Getting Started

This guide will help you set up the Aegis RFQ platform for development and production use.

## Prerequisites

Before you begin, ensure you have the following installed:

- **DAML SDK 2.10.2+**: [Installation Guide](https://docs.daml.com/getting-started/installation.html)
- **Bun**: Fast JavaScript runtime (preferred) or Node.js 18+
- **Git**: Version control

## Installation

### 1. Clone the Repository

```bash
git clone https://github.com/your-org/aegis-rfq.git
cd aegis-rfq
```

### 2. Set Up DAML Contracts

```bash
# Build DAML contracts
daml build

# Run tests
daml test

# Start DAML ledger (sandbox)
daml start
```

This will start:
- DAML Ledger on port `6865`
- JSON API on port `7575`
- Navigator on port `7500`

### 3. Set Up Backend API

```bash
cd backend

# Install dependencies
bun install

# Copy environment file
cp .env.example .env

# Configure your .env file
# Edit .env with your settings

# Run database migrations
bun run db:migrate

# Start development server
bun run dev
```

The backend API will be available at `http://localhost:8000`

### 4. Set Up Frontend

```bash
cd frontend/apps/web

# Install dependencies
bun install

# Copy environment file
cp .env.example .env.local

# Configure your .env.local file
# Edit .env.local with your settings

# Start development server
bun run dev
```

The frontend will be available at `http://localhost:3000`

## Environment Configuration

### Backend (.env)

```bash
# DAML Configuration
DAML_JSON_API_URL=http://localhost:7575
DAML_WS_URL=ws://localhost:7575/v1/stream/query
DAML_LEDGER_ID=sandbox
DAML_APPLICATION_ID=aegis-rfq-platform

# JWT Configuration
JWT_SECRET=your-secret-key-here
JWT_EXPIRY=3600

# Database
DATABASE_URL=./aegis.db

# Business Rules
MIN_COLLATERAL_RATIO=1.2
MAX_INTEREST_RATE=0.25
DEFAULT_RFQ_EXPIRY_DAYS=7

# Rate Limiting
RATE_LIMIT=100
RATE_LIMIT_WINDOW=900000
```

### Frontend (.env.local)

```bash
# API Configuration
NEXT_PUBLIC_API_URL=http://localhost:8000
NEXT_PUBLIC_WS_URL=ws://localhost:7575

# DAML Configuration
NEXT_PUBLIC_DAML_JSON_API_URL=http://localhost:7575
NEXT_PUBLIC_DAML_LEDGER_ID=sandbox
```

## Development Workflow

### 1. DAML Development

```bash
# Build contracts
daml build

# Run tests
daml test

# Generate TypeScript bindings
daml codegen js
```

### 2. Backend Development

```bash
cd backend

# Development with hot reload
bun run dev

# Generate JWT token for testing
bun run generate-token Alice

# Run database migrations
bun run db:migrate

# View database
bun run db:studio
```

### 3. Frontend Development

```bash
cd frontend/apps/web

# Development with Turbopack
bun run dev

# Build for production
bun run build

# Start production server
bun run start
```
